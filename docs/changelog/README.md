# Technical Changelog

**Purpose**: Registro cronológico de todos los avances técnicos del proyecto Pyadra.

---

## Qué va aquí

✅ **SÍ documentar:**
- Cambios de arquitectura
- Migraciones importantes
- Refactorings significativos
- Mejoras de performance
- Cambios de estructura de URLs
- Auditorías técnicas
- Deployment updates
- Security patches

❌ **NO documentar:**
- Feature requests (va en issues)
- Bug fixes menores
- Cambios de copy/texto
- Ajustes de diseño visual

## Estructura de Archivos

### Nomenclatura:
```
YYYY-MM-DD_nombre-descriptivo.md

Ejemplos:
2026-04-06_url-migration-galaxy.md
2026-04-07_project-audit-cleanup.md
2026-03-31_on-demand-delivery.md
```

### Formato de Documento:
```markdown
# [Título del Cambio]

**Date**: YYYY-MM-DD  
**Type**: Architecture | Migration | Refactor | Performance | Security  
**Status**: Completed | In Progress | Planned

## Summary
[1-2 párrafos explicando qué se hizo y por qué]

## Changes
- Lista de cambios específicos

## Impact
- Qué afecta
- Qué se rompe (breaking changes)
- Qué mejora

## Migration Steps (si aplica)
1. Pasos para aplicar el cambio

## Related
- Links a PRs, issues, otros docs
```
